# HealthPulse 기술 제안서

## 1. 프로젝트 배경

디지털 헬스케어 및 질병진단 키트 산업은 빠르게 변화하고 있으며, 시장 동향과 경쟁사 움직임을 신속하게 파악하는 것이 경쟁력의 핵심입니다. 본 프로젝트는 네이버 뉴스를 자동 수집하여 일일 리포트를 생성하고 관계자에게 이메일로 발송하는 시스템을 구축합니다.

---

- 헬스케어 + 실시간 동향의 의미를 잘 전달

## 3. 시스템 아키텍처

### 3.1 전체 흐름

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  뉴스 수집   │ -> │  데이터 처리  │ -> │  리포트 생성  │ -> │  이메일 발송  │
│  (Crawler)  │    │  (Filter/AI) │    │  (Template)  │    │   (SMTP)    │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       ↑                                                        │
       └──────────────── 스케줄러 (매일 자동 실행) ─────────────────┘
```

### 3.2 모듈별 상세

#### 뉴스 수집 (Crawler)
```python
# 검색 키워드 예시
keywords = [
    "디지털헬스케어", "진단키트", "체외진단",
    "의료기기", "바이오마커", "AI진단",
    "원격의료", "디지털치료제", "웨어러블",
    "FDA승인", "식약처", "임상시험"
]
```

#### 데이터 처리 (Processor)
- 중복 기사 제거 (제목/본문 유사도 분석)
- 관련성 필터링 (헬스케어 도메인 특화)
- AI 기반 기사 요약 (3-5줄)
- 카테고리 자동 분류
- 중요도 점수 산정

#### 리포트 생성 (Reporter)
- HTML 템플릿 기반 뉴스레터 형식
- 섹션별 구성:
  - 오늘의 주요 뉴스 (TOP 5)
  - 카테고리별 뉴스 (규제/시장/기술/경쟁사)
  - 키워드 트렌드 분석
- PDF 첨부 옵션

#### 이메일 발송 (Mailer)
- 수신자 그룹별 맞춤 발송
  - 경영진: 요약 중심
  - 연구팀: 기술/임상 중심
  - 마케팅: 시장/경쟁사 중심
- 발송 이력 관리
- 실패 시 재시도 로직

## 4. 뉴스 수집 방법 비교

### Option A: 네이버 검색 API (권장)

| 항목 | 내용 |
|------|------|
| 방식 | 공식 Open API |
| 장점 | 안정적, 법적 문제 없음 |
| 단점 | 일일 25,000건 제한 |
| 비용 | 무료 |
| 신청 | https://developers.naver.com |

```python
# API 호출 예시
import requests

url = "https://openapi.naver.com/v1/search/news.json"
headers = {
    "X-Naver-Client-Id": "YOUR_CLIENT_ID",
    "X-Naver-Client-Secret": "YOUR_CLIENT_SECRET"
}
params = {
    "query": "디지털헬스케어",
    "display": 100,
    "sort": "date"
}
response = requests.get(url, headers=headers, params=params)
```

### Option B: 웹 크롤링

| 항목 | 내용 |
|------|------|
| 방식 | BeautifulSoup / Selenium |
| 장점 | 제한 없음, 상세 정보 수집 가능 |
| 단점 | robots.txt 준수 필요, 유지보수 비용 |
| 비용 | 서버 비용만 |

### 권장사항
- **1차**: 네이버 검색 API 사용
- **2차**: API 한도 초과 시 크롤링 병행

## 5. 기술 스택

### 5.1 백엔드

| 구성요소 | 기술 | 선정 이유 |
|---------|------|----------|
| 언어 | Python 3.10+ | 크롤링/AI 라이브러리 풍부 |
| 웹 프레임워크 | FastAPI | 비동기 지원, 자동 문서화 |
| 스케줄러 | APScheduler | 경량, Python 네이티브 |
| ORM | SQLAlchemy | 유연한 DB 연동 |

### 5.2 데이터베이스

| 용도 | 기술 | 선정 이유 |
|------|------|----------|
| 메인 DB | PostgreSQL | 안정성, 전문 검색 지원 |
| 캐시 | Redis | 중복 체크, 세션 관리 |

### 5.3 AI/ML

| 용도 | 기술 | 선정 이유 |
|------|------|----------|
| 기사 요약 | Claude API / OpenAI | 고품질 한국어 요약 |
| 유사도 분석 | Sentence-BERT | 중복 기사 탐지 |

### 5.4 인프라

| 구성요소 | 기술 | 선정 이유 |
|---------|------|----------|
| 배포 | AWS Lambda | 서버리스, 비용 효율 |
| 스케줄링 | CloudWatch Events | Lambda 트리거 |
| 이메일 | AWS SES | 대량 발송, 저비용 |
| 모니터링 | CloudWatch Logs | 통합 로그 관리 |

## 8. 법적/윤리적 고려사항

### 8.1 저작권
- ✅ 기사 제목 + 요약 + 원문 링크 제공
- ❌ 기사 전문 복사 금지
- ✅ 출처 명시 필수

### 8.2 개인정보보호
- 이메일 수신자 동의 필요
- 수신거부 기능 필수 제공
- 개인정보 암호화 저장

### 8.3 네이버 이용약관
- API 사용 시 이용약관 준수
- 크롤링 시 robots.txt 준수
- 과도한 요청 자제 (rate limiting)
